# Invest by Vitalii Khilko — Telegram automation MVP

Система генерирует инвестиционные посты по рынку США из `data/manual_inputs.json`,
создаёт PNG-обложку, отправляет черновик в закрытый Telegram-чат и публикует его
в `@Financebks` только после ручного нажатия кнопки **✅ Опубликовать**.

## Что входит в MVP

- два формата: `morning_brief` и `evening_theme`;
- структурированный JSON от OpenAI;
- отдельный quality-check и служебные предупреждения;
- AI-картинка в редакционном стиле канала с HTML/CSS fallback;
- кнопки публикации, отклонения и двух вариантов переработки;
- файловые черновики, опубликованные записи и JSONL-логи;
- заглушки для будущих X/Twitter, Telegram, Yahoo Finance и Barchart источников;
- два расписания GitHub Actions.

Никакие реальные токены или ключи не должны попадать в репозиторий, логи или
workflow-файлы. Все доступы передаются только через переменные окружения и
GitHub Secrets.

## Ручной редакционный отбор новостей

Основной режим работы — ручной отбор сюжетов в чате согласования:

1. Отправьте боту команду `/news` или `/start`.
2. Выберите подборку для утреннего или вечернего поста.
3. Бот проверит свежие подключённые источники и отправит новости отдельными карточками.
4. Нажмите `✅ Принять новость` под подходящим сюжетом.
5. Бот перепишет выбранную новость в формате канала, выполнит факт-чек и создаст новую
   тематическую AI-иллюстрацию.
6. Готовый текст и картинка придут одним сообщением с кнопкой `✅ Опубликовать`.

Настройки отбора вручную редактируются в `data/editorial_policy.json`. Там можно
изменить веса источников, силу катализаторов, список ключевых макро/секторных
слов, штрафы за перегретые темы вроде `NVDA`, минимальный проходной балл и
правила редакционного фокуса. Чем выше итоговый `editorial_score`, тем выше
новость попадает в подборке.

Команда `/news` работает только в закрытом чате согласования. Автоматические расписания
утреннего и вечернего постов включены; workflows `morning_post.yml` и `evening_post.yml`
также можно запускать вручную через GitHub Actions.

Визуальный стиль картинок настраивается в `data/visual_policy.json`. Этот файл
задаёт редакционную логику для изображений: рыночный терминал, продуктовый
контекст компании, макро-сцена, карта экосистемы, структура сделки/потоков или
журнальная иллюстрация. Генератор специально избегает однотипных плоских
обложек и меняет композицию, ракурс или формат при каждом нажатии
**🖼 Ещё вариант**. AI-иллюстрации намеренно генерируются без текста, цифр,
тикеров на самой картинке и выдуманных логотипов: это защищает от битых букв,
фейковых брендов и недостоверных визуальных деталей. Настоящие логотипы
допустимо добавлять только из проверенных официальных бренд-материалов.

Обязательные переменные MVP: `OPENAI_API_KEY`, `TELEGRAM_BOT_TOKEN`,
`TELEGRAM_REVIEW_CHAT_ID`, `TELEGRAM_CHANNEL_ID`. Без них рабочая команда
завершится понятной ошибкой.

Остальные переменные опциональны. Если одна или несколько из них отсутствуют,
система записывает предупреждение в `data/logs/`, использует
`data/manual_inputs.json` и продолжает отправку черновика в review-чат.

## 4. Telegram-настройка

### Как получить `TELEGRAM_REVIEW_CHAT_ID`

1. Добавьте `@Invest_by_VitaliiKhilko_bot` в закрытый чат согласования.
2. Напишите сообщение в чат.
3. Локально вызовите Bot API `getUpdates` с токеном только через окружение или
   используйте диагностический скрипт, который не печатает токен.
4. Найдите отрицательный `chat.id`. У супергрупп он обычно начинается с `-100`.

### Канал и права бота

Основной канал: [@Financebks](https://t.me/Financebks).

Добавьте бота администратором в канал и разрешите публикацию сообщений. Также
добавьте его в чат согласования, чтобы он мог отправлять сообщения и кнопки.
`TELEGRAM_CHANNEL_ID` можно задавать как публичный username `@Financebks`.

### Перевыпуск bot token

Если токен раскрыт, откройте `@BotFather`, выберите бота, выполните отзыв
текущего токена и создайте новый. После этого обновите только локальный `.env`
и GitHub Secret `TELEGRAM_BOT_TOKEN`.

## 5. Запуск MVP локально

Отредактируйте `data/manual_inputs.json`, затем в одном терминале запустите
постоянный обработчик кнопок:

```bash
python -m app.main bot
```

Во втором терминале создайте тестовый черновик:

```bash
python -m app.main generate --type morning_brief
```

Для вечерней темы:

```bash
python -m app.main generate --type evening_theme
```

Черновик и картинка появятся в `data/drafts/`, после публикации запись также
сохранится в `data/published/`, а события и ошибки — в `data/logs/`.

## 6. Как работает согласование

Бот отправляет публикуемый пост одним сообщением: картинка и текст в подписи.
Под ним доступны основные кнопки:

- **✅ Опубликовать** — копирует единое сообщение в `@Financebks`;
- **🔁 Новый вариант текста** — повторно вызывает OpenAI и создаёт новый черновик;
- **🖼 Новый вариант картинки** — создаёт альтернативную обложку с кнопками
  **Принять картинку** и **Отклонить картинку**;
- **❌ Отклонить** — сохраняет статус `rejected`.
- **📰 Другая новость** — создаёт новый сюжет по другому свежему сигналу.

После принятия альтернативная картинка заменяет обложку в основном черновике.
Из-за ограничения Telegram подпись публикуемого поста не превышает 1000 символов.

Quality-check проверяет длину, дату, тикеры, запрещённые формулировки, прямые
рекомендации, обещания доходности, маркировку слухов и `risk_flags`. Не прошедший
проверку пост всё равно приходит в чат, но никогда не публикуется автоматически.
В MVP автоматической публикации вообще нет: любое размещение требует кнопки.

Перед генерацией система отбрасывает все сигналы без текущей даты по Москве и
сигналы с прошедшей датой. Если подтверждённых свежих сигналов из разрешённых
источников нет, бот отправляет предупреждение в чат согласования и не создаёт
пост. Это защищает канал от выдуманных или устаревших новостей.

Yahoo discovery сканирует широкий рынок по тематическим запросам: макро и
ставки, отчётность, M&A, инсайдерская активность, опционы, банки, медицина,
энергетика, промышленность, потребительские компании, сырьё, недвижимость,
коммунальные компании, технологии и small caps. AI и мегатехи не получают
автоматического приоритета: главный сюжет выбирается по значимости свежих данных.

## 7. GitHub Actions

Workflow `morning_post.yml` запускается ежедневно в `07:10 UTC`, то есть в
`10:10 MSK`. `evening_post.yml` запускается ежедневно в `14:30 UTC`, то есть в
`17:30 MSK`. `source_health.yml` ежедневно проверяет источники в `06:40 UTC`, то
есть в `09:40 MSK`.
Все workflow также можно запустить вручную во вкладке **Actions**.

`draft_watchdog.yml` независимо проверяет расписание каждые 10 минут. Если
GitHub полностью пропустил основной cron, watchdog запускает отсутствующий
утренний или вечерний черновик: утренний после `07:20 UTC` / `10:20 MSK`,
вечерний после `14:40 UTC` / `17:40 MSK`. Повторная отправка исключается
проверкой уже успешных, ожидающих и выполняющихся запусков за текущий день;
если запуск был с ошибкой, watchdog повторит попытку не чаще одного раза в час.

Workflow `process_telegram_callbacks.yml` постоянно держит обработчик кнопок
онлайн. Каждый экземпляр перед
завершением самостоятельно запускает следующий, поэтому работа кнопок не зависит
только от ненадёжного GitHub cron. Watchdog также восстанавливает обработчик,
если он полностью остановился. При временной ошибке генерации кнопки исходного
черновика автоматически восстанавливаются.
Для мгновенного отклика запустите `python -m app.main bot` на постоянно
работающем сервере. Облачная обработка кнопок не зависит от временных файлов:
публикация копирует согласованную обложку и текст непосредственно из review-чата.

## 8. Расширение источников

- **X/Twitter:** адаптер читает приоритетные аккаунты через X API v2. Нужен
  рабочий `X_BEARER_TOKEN` с доступом к чтению пользователей и постов.
- **Telegram:** адаптер читает `krasilnikovbroker` и `utexplatform` через
  Telethon/MTProto. Получите `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` на
  `https://my.telegram.org/apps`, добавьте их локально в `.env`, затем один раз
  запустите `python scripts/telegram_string_session.py`. После подтверждения
  входа добавьте выведенный `TELEGRAM_SESSION_STRING` в GitHub Secrets. Никогда
  не публикуйте session string и не сохраняйте его в репозитории.
- **Yahoo Finance:** адаптер автоматически собирает свежие новости и рыночный
  snapshot. Отдельный ключ сейчас не требуется.
- **Barchart:** адаптер получает котировки watchlist через Barchart OnDemand.
  Нужен действующий `BARCHART_API_KEY` с доступом к `getQuote`.
- **Oninvest:** источник остаётся отключённым до получения официальной
  документации API и `ONINVEST_API_KEY`. HTML-скрейпинг сайта намеренно не
  используется как ненадёжный источник.

Каждый сигнал должен содержать источник, суть, тикеры/секторы, воздействие,
силу, горизонт, дату и тип катализатора. Перед подключением источника проверьте
его условия использования и лицензионные ограничения.
